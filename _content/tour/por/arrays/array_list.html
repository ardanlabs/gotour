• Caches de CPU funcionam armazenando a memória principal em linhas de cache.
  • Linhas de cache são de tamanho 32 ou 64 bytes dependendo do hardware.
  • Núcleos de processadores não acessam diretamente a memória principal. Eles tendem a ter acesso somente ao seu cache local.
  • Ambas as datas e instruções são armazenados nos caches.
  • Linhas de caches são movidas para L1->L2->L3 â medida que novas linhas de cache precisam ser armazenadas nos caches .
  • Hardware gosta de percorrer dados e instruções de forma linear ao longo das linhas de cache.

  • A memória principal é construída com memória relativamente rápida e barata. Os caches são construídos com memória muito rápida e cara.

  • O acesso à memória principal é incrivelmente lento, por isso precisamos da cache.

    • Acessar um byte na memória principal fará com que uma linha de cache inteira seja lida e armazenada em cache.
    • Escrever um byte em uma linha de cache requer que a linha de cache inteira seja escrita.

  • Pequeno = Rápido

    • Código compacto, bem localizado e que cabe na cache é o mais rápido.
    • Estruturas de dados compactas que cabem na cache são as mais rápidas.
    • Travessias que acessam apenas dados em cache são as mais rápidas.

  • Padrões de acesso previsíveis são importantes.

    • Sempre que for possível, você deseja empregar uma travessia linear de array.
    • Fornecer padrões regulares de acesso à memória.
    • O hardware pode fazer previsões melhores sobre a memória necessária.

  • Falhas de cache podem resultar em falhas de cache do TLB também.

    • Cache de traduções de um endereço virtual para um endereço físico.
    • Esperar pelo sistema operacional nos dizer onde está a memória.
